# E-Commerce Fraud Analysis (SQL Project)

## Overview
This project analyzes mock e-commerce transaction data to identify fraud patterns using SQL.

The goal is to demonstrate:
- Fraud rate calculation
- Pattern identification
- Risk segmentation
- Business recommendations

## Dataset Structure
The dataset includes:
- Transaction ID
- User ID
- Transaction Date
- Amount
- Country
- Payment Method
- Fraud Indicator (0 = Legit, 1 = Fraud)

## Key Analysis Performed
- Overall fraud rate
- Fraud rate by country
- Fraud rate by payment method
- High-value transaction analysis
- Repeat fraud user detection

## Tools Used
- SQL (Basic SELECT, WHERE, GROUP BY, HAVING)
- GitHub for documentation

## Key Insights
- Fraud rates vary by geography
- High-value transactions show elevated fraud risk
- Certain payment methods demonstrate higher fraud rates
